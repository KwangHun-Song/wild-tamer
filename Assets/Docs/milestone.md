# Wild Tamer Like - 마일스톤

## Phase 1: 프로젝트 베이스

프로젝트의 기반이 되는 공통 모듈들을 asmdef 기반으로 의존성을 분리하여 구축한다.

### 1.1 프로젝트 구조 설정

- [ ] asmdef 기반 모듈 분리 구조 설계 및 폴더 구성
- [ ] 공통 유틸리티 모듈 (Extensions, Helpers 등)

### 1.2 데이터 관리

- [ ] 데이터 저장/불러오기 시스템 (로컬 Save/Load)
- [ ] 게임 데이터 테이블 로딩 (ScriptableObject 또는 JSON 기반)

### 1.3 UI 프레임워크

- [ ] 페이지(Scene/View) 전환 시스템
- [ ] 팝업 매니저 (열기/닫기/스택 관리)

### 1.4 사운드

- [ ] 사운드 매니저 (BGM, SFX 재생/정지/볼륨 조절)

### 1.5 오브젝트 풀링

- [ ] 오브젝트 풀 시스템 (다수 개체 생성/회수 최적화)

### 1.6 이벤트 시스템

- [ ] 전역 이벤트 버스 (시스템 간 느슨한 결합)

### 1.7 리소스 로딩

- [ ] Addressables 또는 Resources 기반 에셋 로딩 관리

## Phase 2: 코어 시스템

게임의 핵심 플레이 루프를 구성하는 시스템들을 구현한다.

### 2.1 플레이어 이동 및 입력

- [ ] 플레이어 캐릭터 이동 (조이스틱 또는 키보드)
- [ ] 쿼터뷰 카메라 플레이어 추적 시스템

### 2.2 군집 및 부대 시스템

- [ ] 길찾기 알고리즘 (NavMesh 미사용, 직접 구현)
- [ ] 부대원이 플레이어를 따라다니는 군집 이동
- [ ] 대형 유지 및 부드러운 이동 처리

### 2.3 월드맵

- [ ] 타일맵 또는 프로시저럴 기반 맵 생성
- [ ] 지형 및 장애물 배치

### 2.4 몬스터

- [ ] 몬스터 데이터 구조 설계 (종류, 스탯, 공격 패턴)
- [ ] 일반 몬스터 3종 구현 (종별 이동 패턴, 공격 사거리 차별화)
- [ ] 보스 몬스터 2종 구현 (장판, 돌진 등 특수 스킬 패턴)
- [ ] 몬스터 랜덤 스폰 시스템
- [ ] 야생 몬스터 부대 형태 군집 이동

### 2.5 자동 전투 시스템

- [ ] 적 감지 및 자동 타겟팅
- [ ] 아군/적 자동 교전 로직
- [ ] 타격 판정 및 데미지 처리
- [ ] 체력 시스템 및 사망 처리

### 2.6 테이밍

- [ ] 몬스터 처치 시 확률 기반 테이밍 판정
- [ ] 테이밍 성공 시 부대 합류 처리
- [ ] 테이밍 연출 (시각적 피드백)

### 2.7 전투 연출

- [ ] 공격/피격 시 역경직 (히트스톱)
- [ ] 화면 흔들림 (카메라 셰이크)
- [ ] 타격 이펙트 및 사운드

### 2.8 전장의 안개 (Fog of War)

- [ ] 미탐색 지역 어둡게 처리
- [ ] 플레이어 이동에 따른 시야 공개

### 2.9 미니맵

- [ ] 미니맵 UI
- [ ] 실시간 탐색 현황 반영
- [ ] 플레이어/아군/적 위치 표시
